🎥 PRACTICA 1 – Sincronización con Rsync
# Primero instalo rsync en ambos servidores
sudo dnf install rsync -y

# En el servidor principal (192.168.0.199) creo la carpeta a sincronizar
mkdir /home/nolberto/syncfolder
cd /home/nolberto/syncfolder

# Creo 100 archivos de prueba
for i in {1..100}; do touch file$i.txt; done

# Verifico que los 100 archivos fueron creados
ls | wc -l

# Genero clave SSH para conexión sin contraseña
ssh-keygen

# Copio la clave al servidor secundario (192.168.0.198)
ssh-copy-id nolberto@192.168.0.198

# Realizo una sincronización manual de prueba
rsync -avz /home/nolberto/syncfolder/ nolberto@192.168.0.200:/home/nolberto/syncfolder/

# Creo un script para automatizar la sincronización
nano /home/nolberto/sync.sh

Contenido del script:

#!/bin/bash
rsync -avz /home/nolberto/syncfolder/ nolberto@192.168.0.198:/home/nolberto/syncfolder/
# Doy permisos de ejecución
chmod +x /home/nolberto/sync.sh

# Programo el cron para que se ejecute cada minuto
crontab -e

Agregar:

* * * * * /home/nolberto/sync.sh
# Creo un archivo nuevo para probar el cron
touch /home/nolberto/syncfolder/prueba_cron.txt

Después de 1 minuto verificas en 192.168.0.198.

 PRACTICA 2 - CLUSTER HA (PCS)


OBJETIVO: Implementar cluster de alta disponibilidad con Pacemaker y
Corosync.

—- EN AMBOS SERVIDORES —-

sudo dnf install pcs pacemaker corosync -y sudo systemctl enable pcsd
sudo systemctl start pcsd sudo passwd hacluster

—- DESDE SERVER 1 —-

sudo pcs host auth 192.168.0.199 192.168.0.198 -u hacluster -p
TU_PASSWORD sudo pcs cluster setup mi_cluster 192.168.0.199
192.168.0.198 –force sudo pcs cluster enable –all sudo pcs cluster start
–all sudo pcs status

Crear IP Virtual del cluster

sudo pcs resource create VIP ocf:heartbeat:IPaddr2 ip=192.168.0.210
cidr_netmask=24 op monitor interval=5s

Desactivar stonith (solo laboratorio)

sudo pcs property set stonith-enabled=false sudo pcs status

Probar IP virtual

ping 192.168.0.210

Simular falla

sudo reboot

============================== PRACTICA 3 - KEEPALIVED (VRRP)
==============================

OBJETIVO: Implementar alta disponibilidad con IP virtual usando VRRP.

—- EN AMBOS SERVIDORES —-

sudo dnf install keepalived -y

Editar: sudo nano /etc/keepalived/keepalived.conf

—- SERVER 1 (MASTER) —-

global_defs { router_id SERVER1 }

vrrp_instance VI_1 { state MASTER interface ens160 virtual_router_id 51
priority 100 advert_int 1

    authentication {
        auth_type PASS
        auth_pass 1234
    }

    virtual_ipaddress {
        192.168.0.220/24
    }

}

SERVER 2 (BACKUP)

Cambiar: router_id SERVER2 state BACKUP priority 90

FINALIZAR

sudo firewall-cmd –zone=public –add-protocol=vrrp –permanent sudo
firewall-cmd –reload sudo systemctl enable keepalived sudo systemctl
start keepalived

Verificar IP virtual

ip a ping 192.168.0.220

Simular falla

nmcli con down ens160 nmcli con up ens160
